# Issue 32.1: Remove Hardcoded Sample Data from Production; Add Test Fixtures

**Status**: Open
**Priority**: P2 - Medium
**Created**: 2026-03-07
**Category**: Code Quality / Test Infrastructure
**GitHub Issue**: [#427](https://github.com/ezigus/zpod/issues/427)

## Description

Eliminate all hardcoded podcast and episode data from production source files and replace them with proper test fixtures in `Packages/TestSupport`. This separates production code from fixture data, ensures the app shows a true empty state for new users, and gives test authors a well-known, reusable fixture set.

## Problem Statement

Hardcoded sample podcasts and episodes are scattered across production source files:

| File | Symbol | Impact |
|------|--------|--------|
| `Packages/LibraryFeature/.../ContentView.swift` | `samplePodcasts` in `loadData()` | Library always shows phantom data |
| `zpod/ContentViewBridge.swift` | `PlaceholderPodcastData` enum (~lines 78–139) | Placeholder podcasts in non-iOS targets |
| `zpod/ContentViewBridge.swift` | `LibraryPlaceholderView.samplePodcasts` (~lines 346–350) | Phantom library entries |
| `zpod/ContentViewBridge.swift` | `EpisodeListPlaceholderView.sampleEpisodes` (~lines 415–419) | Phantom episode entries |

**Note**: The `ContentView.swift` `samplePodcasts` will already be removed by Issue 27.1.9. Issue 32.1 verifies that removal and cleans up the remaining traces in `ContentViewBridge.swift`.

## Acceptance Criteria

- [ ] No hardcoded podcast or episode data in any production source file
- [ ] `Packages/TestSupport/Sources/TestSupport/Fixtures/PodcastFixtures.swift` exists with at least three well-known test podcasts (e.g., Swift Talk, Swift Over Coffee, ATP) importable in test `setUp()` calls
- [ ] `Packages/TestSupport/Sources/TestSupport/Fixtures/EpisodeFixtures.swift` exists with sample episodes tied to the fixture podcasts
- [ ] Any existing tests that relied on hardcoded sample data now use fixtures in their Given setup phase
- [ ] Production app with zero subscriptions shows empty state (not phantom sample podcasts)
- [ ] CI passes with no regressions (`./scripts/run-xcode-tests.sh`)

## Implementation Notes

### Empty State for ContentViewBridge

`ContentViewBridge.swift` serves non-iOS targets where `LibraryFeature` is not importable. Replace the placeholder structs with minimal empty-state views that display a user-facing message, matching the empty-state style established in 27.1.9.

### Fixture Design Principles

- Use stable, deterministic values (no timestamps or UUIDs) so fixtures produce consistent snapshots
- Keep URLs in the `example.com` domain — never reference real podcast feeds in fixtures
- Fixtures are value types seeded in test `setUp()` — they do not persist to disk unless the test explicitly calls the repository

# Issue 27.1.9: Wire Library View to Live Podcast Repository Data

**Status**: Complete
**Priority**: P1 - High
**Created**: 2026-03-07
**Category**: Feature / UI Wiring
**GitHub Issue**: [#426](https://github.com/ezigus/zpod/issues/426)

## Description

Surface the persistent `PodcastManaging` data in the Library tab. Currently `ContentView.loadData()` populates the library with a hardcoded `samplePodcasts` array instead of fetching from the live repository that was wired in Issue 27.1.2. This means podcasts added via Discover never appear in the Library, and the app does not persist subscription state across launches from the user's perspective.

## Problem Statement

**Current State** (`Packages/LibraryFeature/Sources/LibraryFeature/ContentView.swift`, `loadData()` ~lines 778â€“792):
```swift
private func loadData() {
    // âťŚ Hardcoded sample data â€” ignores the injected podcastManager
    self.podcasts = samplePodcasts
}
```

`podcastManager: PodcastManaging` is already injected into `ContentView`, and `PlaylistTabView` already calls `podcastManager.all()` successfully (~line 167). The Library just never wires to it.

**Impact**:
- Podcasts added via Discover tab do not appear in Library
- Library always shows the same hardcoded sample entries
- Persistence infrastructure (Issue 27.1.2) is wired but invisible to the user
- No empty state shown when a user has no subscriptions

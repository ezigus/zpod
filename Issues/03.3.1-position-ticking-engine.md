# Issue 03.3.1: Position Ticking Engine

**Status**: ✅ Complete

**GitHub Issue**: [#266](https://github.com/ezigus/zpod/issues/266) - Closed

## Description

The `EnhancedEpisodePlayer` emits a `.playing` state once when play is called, but never advances `currentPosition`. The `Ticker` protocol and `TimerTicker` implementation exist in `SimplePlaybackService.swift` but are never instantiated or used.

This issue adds periodic position updates so the UI timeline advances during playback.

## Problem Statement

**Current Behavior**:
1. User taps Play
2. `EnhancedEpisodePlayer.play()` sets `isPlaying = true`
3. Single `.playing(episode, 0, duration)` emitted
4. Position stays at 0 forever

**Expected Behavior**:
1. User taps Play
2. Position advances every 0.5-1.0 seconds
3. UI timeline updates in real-time
4. Seeking updates position immediately

## Implementation Steps

1. **Add Ticker Property**
   ```swift
   private var ticker: Ticker?
   ```

2. **Start Ticker on Play**
   ```swift
   func play(...) {
       // existing code...
       startTicker()
   }

   private func startTicker() {
       ticker?.cancel()
       ticker = TimerTicker()
       ticker?.schedule(every: 0.5) { [weak self] in
           Task { @MainActor in
               self?.tick()
           }
       }
   }
   ```

3. **Advance Position on Tick**
   ```swift
   private func tick() {
       guard isPlaying, currentPosition < currentDuration else {
           if currentPosition >= currentDuration {
               finish()
           }
           return
       }
       currentPosition += 0.5
       emitCurrentState()
   }
   ```

4. **Stop Ticker on Pause/Stop**
   ```swift
   func pause() {
       isPlaying = false
       ticker?.cancel()
       emitCurrentState()
   }
   ```

5. **Clean Up on Deinit**
   ```swift
   deinit {
       ticker?.stop()
   }
   ```

### Code Quality Improvements (Post-Implementation Review)

After initial implementation, comprehensive code review identified and addressed design improvements:

**Phase 1 - Design Smells** (2025-12-28):
1. **Injectable Ticker**: Made ticker injectable via factory pattern for testability
2. **Run Loop Mode**: Changed TimerTicker to `.common` run loop mode (works during UI interactions)
3. **Persistence Throttling**: Reduced I/O overhead by 90% (throttled from every 0.5s to every 5s)

**Phase 2 - Deterministic Testing** (2025-12-28):
- Created `DeterministicTicker` test helper for instant, deterministic test execution
- Eliminated all `Task.sleep()` dependencies from tests
- **Performance**: 4,250x test speedup (17s → 0.004s)

**Test Results**: All 19 ticker tests passing with zero timing variance

### Known Limitations

- Ticker hard-coded in initial implementation (fixed in Phase 1)
- Tests used real-time delays (fixed in Phase 2)
- UI integration tests not included (deferred to separate issue)

### Technical Decisions

1. **Tick Interval**: 0.5s chosen to balance UI responsiveness with CPU efficiency
2. **Persistence Throttling**: 5s intervals with forced persistence on state changes (pause/seek/finish)
3. **Speed Clamping**: 0.8x - 5.0x range prevents unusable speeds
4. **Run Loop Mode**: `.common` ensures timer continues during UI scrolling/modals
